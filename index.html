Compilers & Code Generation
Siphia Compiler

Sophia Language

Project Description

Sophia is an object-oriented Java-like programming language which supports inheritance. Sophia compiler has been implemented in four phases using Java in IntelliJ IDEA.

It has been developed as a student project for Compiler Design and Programming Languages course.

Phase I, Lexical and Syntax Analyzer: Sophia's grammar was implemented using ANTLR and as well as a parser to create abstract syntax tree (AST) of input program.

Phase II, Name Analyzer: The name analyzer of the language was implemented to gather information about classes, fields, methods, local variables, and scopes of program and finding name analysis errors like undefined variables or redefinition.

Phase III, Type Analyzer: A type checker was implemented to check the types of variables in input program, finding relevant errors based on Sophia's documentation and setting initial value for various types.

Phase IV, Code Generation: The machine code was generated by converting Sophia's code to jasmin as a middle layer. Jasmin assembler converts its codes to class files which can be executed by JVM later.

Operating Systems
XV6 Operating System

Working woth XV6

Project Description

XV6 is a teaching operating system which was edited and improved as a student project for Operating Systems course. Having implemented various features, I have been familiar with technical aspects of operating systems.

Introduction to Xv6 OS: I got familiar with Xv6, its execution, debugging, improving its console by adding some features, and implementing new program to copy text data from a file to another one.

XV6 System Calls: Several new system calls were implemented and added to the OS which enable XV6 to run commands out of the directory in which their execution file exists, sleep a process for an arbitrary period using Xv6's ticks, and obtain current process's pid and its children and grandchildren.

Process Scheduling: A multilevel feedback queue scheduler (MFQ) was implemented which includes lottery, HRRN, and SRPF methods as its scheduling levels in Xv6.

Process Synchronization Mechanisms: Two synchronization mechanisms were added to prevent out-of-order execution of processes, and use mutex more than once in recursive programs.
